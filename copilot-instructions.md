# Copilot Instructions

Apply a BMAD-style workflow in this repository.

Rules:
- Treat `docs/bmad/prd-template.md` as the source of product intent.
- Treat `docs/bmad/architecture-template.md` as the source of implementation boundaries.
- Do not jump into code until the current task is linked to a story or explicit user request.
- Keep outputs grounded in approved documentation and explicit business rules.
- Favor small changes, clear acceptance criteria, and visible quality gates.

When helping with the PPT/PDF MVP:
- Preserve corporate standards through templates and validation rules.
- Prefer deterministic generation steps over unconstrained agent behavior.
- Surface assumptions when a source document or standard is missing.
