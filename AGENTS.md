# BMAD XVOR Agents

## Purpose

This repository uses a BMAD-style flow for a documentation-to-PPT/PDF MVP.

## Operating Rules

1. Start with product intent before implementation details.
2. Convert requirements into architecture before writing code.
3. Break delivery into small, reviewable stories.
4. Require a QA gate before marking work complete.
5. Ground generated content in source documents and template rules.

## Project Context

- Primary product goal: create PowerPoint and PDF outputs from approved documentation.
- Quality goal: outputs must meet corporate structure and formatting standards.
- Collaboration goal: teammates contribute source documents through GitHub and review generated outputs through pull requests.

## Expected Artifacts

- PRD
- Architecture note
- Story backlog
- QA gate checklist
- Demo-ready MVP output
